What Is Company Loop?

Company Loop is a persistent operational intelligence system for a catering and event business.

It is NOT:

  • A chatbot
  • A dashboard
  • A workflow library
  • A collection of disconnected automations

It IS:

  • A continuously running loop that maintains continuity across time
  • A standing operational presence that watches, filters, and acts within bounded authority
  • An intelligence layer that converts raw business activity into bounded judgment
  • A system that surfaces only what has earned the right to matter

Core Components

Master Loop

The persistent organism. Always on, maintaining continuity across sessions. Holds corridor states, open obligations, and memory relationships.

Sub-Loops

Bounded episodes of live work. Open on triggers, must close through lock/refusal/timeout/handoff. Temporary by design.

Medic

Self-healing health monitoring. Runs every hour to check system health, attempt recovery, and escalate critical issues.

Watchdog

Monitors the medic itself. Simple by design - verifies medic ran within expected window. Runs every 30 minutes.

The Nine

Internal documentary substrate. Nine files that define identity, memory, tools, and obligations. The textual spine of the organism.

Corridors

Bounded domains of operational concern. 8 corridors: Sales, Operations, Finance, Partnerships, Workforce, Procurement, Marketing, Executive.

Core Operating Doctrine

These nine principles govern every Company Loop deployment. They are non-negotiable.

  • 1. Persistence

    The loop is always on. It carries identity, memory, active obligations, and monitoring duties across time. It does not wake up as a stranger to yesterday.

  • 2. Continuity Through the Master Loop

    The master loop holds state. It maintains memory relationships, active corridor definitions, heartbeat obligations, and the current shape of what is open, settled, deferred, or awaiting action.

  • 3. Bounded Activation

    Sub-loops open only on defined triggers. The system does not open active work merely because it can. Every live cycle must be traceable to an opening condition.

  • 4. Mandatory Closure

    Every sub-loop must close. No sub-loop is permitted to linger indefinitely. A sub-loop may end in lock, refusal, timeout, or handoff - but it must end.

  • 5. Writeback

    Everything significant writes back into state. A loop that acts without writeback does not learn. The discipline of writeback is what allows the loop to mature rather than merely recur.

  • 6. Filtration Before Surfacing

    The human sees only what survives filtering. The loop exists to spare the human from raw informational pressure by performing the work of noticing, narrowing, comparing, ranking, and eliminating.

  • 7. Refusal is Valid

    Refusal is a valid and often intelligent outcome. The system is not obligated to produce an answer merely because a trigger occurred. Refusal protects from counterfeit certainty.

  • 8. Earned Memory

    Memory promotion is earned, not automatic. The loop records widely but promotes selectively. A loop that remembers everything equally well understands nothing proportionally.

  • 9. Self-Maintenance as First-Class Function

    Self-healing is part of the product. The medic exists because a persistent system that cannot inspect and recover itself is only temporarily persistent.

The Four Lawful Exits

Every sub-loop must close through one of these four paths:

Lock (Promotion)

Successful closure where a conclusion, recommendation, action, or output has survived filtering and is allowed to write upward into state or outward to the human.

Refusal

Explicit closure where the lawful result is NOT to promote. The branch closes because action or promotion would be unsound, premature, unauthorized, or structurally weak.

Timeout

Closure of a sub-loop that did not reach a promotable or refusible conclusion within its allotted operational window. A disciplined acknowledgment that not all processes mature on the current timeline.

Handoff

Closure by transfer rather than completion. Occurs when matter has reached boundary of autonomous authority, exceeded local complexity, or requires human judgment.

Quick Start

# 1. Clone repository
git clone [repository-url]
cd company-loop

# 2. Run bootstrap
bash scripts/bootstrap.sh

# 3. Configure credentials
cp .env.example .env
# Edit .env with your API credentials

# 4. Start in shadow mode
python runtime/main.py --mode shadow

# 5. Test single corridor
python runtime/main.py --corridor sales --mode shadow
