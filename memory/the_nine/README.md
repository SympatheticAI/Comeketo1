# The Nine — Internal Documentary Substrate

The Nine is the formal internal file system that makes identity-stable persistence possible.

It is NOT a convenience structure. It is the **textual spine of the organism**.

---

## What The Nine Contains

The Nine defines:

1. **IDENTITY.md** — What the agent is
2. **BOOTSTRAP.md** — How it wakes up
3. **VOICE.md** — How it speaks
4. **RULES.md** — How it behaves
5. **USER.md** — What it knows about the human
6. **TOOLS.md** — What it can reach
7. **MEMORY.md** — What it remembers durably
8. **TODAY.md** — What happened today (working memory)
9. **HEARTBEAT.md** — What it must check proactively

---

## File Discipline

Each file has a **distinct purpose**.

**Do not:**
- Treat The Nine as a dumping ground for every event
- Confuse working memory with durable memory
- Confuse identity with style
- Confuse tool state with operating doctrine
- Confuse heartbeat obligations with general notes

**The Nine is powerful precisely because it is structured.**

---

## When The Nine Is Read

The Nine may be read:
- At startup
- At sub-loop initialization
- During memory review
- During recovery or diagnostic operations

It is the first thing loaded when continuity must be restored.

---

## When The Nine Is Written

The Nine should be written:
- When identity evolves
- When durable memory is promoted
- At end of day (TODAY.md rollover)
- When tools change
- When operating rules are updated
- When user model is refined

**Not every session requires writes to The Nine.**

Most working context belongs in transient state, not in the documentary substrate.

---

## Failure Modes

The Nine fails through:
- **Staleness** — Files not updated when they should be
- **Misfiled information** — Wrong file for the content
- **Contradictory updates** — Files disagree with each other
- **Missing files** — Expected files don't exist
- **Uncontrolled growth** — Files become too large to use effectively
- **Bootstrap omissions** — Critical identity missing at startup
- **Memory promotion drift** — Everything promoted or nothing promoted

The most damaging failure is not disappearance but **conceptual contamination**: when the files still exist but no longer perform distinct roles cleanly.

---

## Monitoring

The Nine is monitored by:
- The **medic** through freshness checks, structural validation, expected-file presence
- **Claude Code** itself during bootstrap (detects conflicts or insufficiency)
